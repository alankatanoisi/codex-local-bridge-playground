Before you run

  1. Bridge running? Paste in Terminal: curl -s http://127.0.0.1:11437/health — expect {"status":"ok"}.
  2. In the right folder? Run the command from the runner repo folder shown above. The target project is separate; set it via --cwd or the Target folder field.
  3. First time in this folder? The runner will ask for --trust-workspace consent the first time you target a new project.
  4. New to this? Start with a read-only preset — it inspects files without changing anything.

Glossary — what do these terms mean?
worktree
A separate checkout of your repo in a different folder, letting you experiment without disturbing your main branch.
ledger
A journal of every event in a run (tool calls, results, model messages). Used by replay/repair to reconstruct or fix runs.
compaction
When the conversation gets too long for the model, the runner summarizes older turns to keep the context window manageable.
fork
Start a new session that shares a parent's history up to a point, then diverges — like a git branch but for agent sessions.
transcript
A JSON-lines file recording every model message and tool call in a run. Useful for debugging or resuming.
trace
A lower-level log of what the runner sent to the bridge, what came back, and internal decisions. Like a flight recorder.
chaos-ok
Acknowledges a risky flag combination (shell + auto-edits + no prompts) you would otherwise be blocked from using.
LSP
Language Server Protocol — the technology behind go-to-definition, find-references, and diagnostics in your editor. The lsp_query tool lets the model use it.

Recovery undo last-run

Every run that edits files records a manifest under .bridge-runner/runs/<run-id>/. Roll a whole run back from its backups with the recovery CLI (run from the playground folder, point --cwd at the project the runner edited):

recovery 
# list recorded runs (newest first)
node bin/local-bridge-undo.js list-runs --cwd /path/to/project

# preview reverting the most recent run (changes nothing)
node bin/local-bridge-undo.js last-run --cwd /path/to/project --dry-run

# revert it (asks to confirm; add --yes in scripts)
node bin/local-bridge-undo.js last-run --cwd /path/to/project

# revert an older run by run id or session id
node bin/local-bridge-undo.js run <run-id|session-id> --cwd /path/to/project

A file changed after the run you are reverting is marked diverged and skipped unless you add --force, so a later run's work is never clobbered by surprise. Old manifests are not auto-deleted — prune them by removing the matching .bridge-runner/runs/<run-id> folder.
