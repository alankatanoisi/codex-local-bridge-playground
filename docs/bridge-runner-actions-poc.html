Bridge Runner Read-Only GitHub Actions POC

A manual self-hosted GitHub Actions workflow that asks the local bridge runner to perform a dry, read-only pass over the playground repository.

This is personal research infrastructure. It is not Anthropic-approved infrastructure, and it is not a production CI/CD system.

What It Does

  • Runs only when Alan manually starts Bridge Runner Read-Only POC from GitHub Actions.
  • Runs only on the configured self-hosted runner.
  • Checks that http://127.0.0.1:11437/ answers.
  • Invokes the local bridge runner with --plan and only read-only tools.
  • Uploads runner JSON, human log, and trace artifacts.

How To Run It

  1. Open the playground repository on GitHub in a browser.
  2. Click the Actions tab.
  3. Click Bridge Runner Read-Only POC.
  4. Click Run workflow.
  5. Leave the default prompt and max_steps value for the first run.
  6. Click the green Run workflow button.

Success Artifacts

Success means the job finishes with a green check mark and these downloadable artifacts appear:

bridge-runner-readonly-poc-json Structured runner result
bridge-runner-readonly-poc-human-log Human-readable run log
bridge-runner-readonly-poc-trace Summary trace and stderr log

Common Failures

  • Could not connect to the local bridge: open VS Code and start Claude Local Bridge.
  • This POC is only intended for the playground repository: the job is in the wrong checkout.
  • BRIDGE_RUNNER_POC_MAX_STEPS must be a number: rerun with a number such as 4.
  • Missing artifacts after an early failure: read the job log first.

Safety Boundaries

  • The workflow uses workflow_dispatch only.
  • The job has contents: read GitHub permissions.
  • The runner command does not include --allow-shell, --accept-edits, or --dont-ask.
  • actions-runner/ is ignored by Git and skipped by runner file traversal tools.
